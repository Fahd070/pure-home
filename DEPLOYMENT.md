# Pure Home — Deployment Guide (Tailscale Edition)

Pure Home Water Filter Maintenance System  
Version 1.0 | Tailscale Private Network Deployment

---

## Project Structure

| Path | Purpose |
|---|---|
| `Desktop\WFM-System-Updated\` | **Source of truth** — all development, builds, and updates originate here |
| `C:\Program Files\Pure Home\` | **Production installation** — generated by the NSIS installer, never edit directly |
| `C:\WFM\` | **Server runtime** — backend service runs from here on the server PC |

> Always make code changes in `Desktop\WFM-System-Updated\`, rebuild, and then re-install.  
> Do **not** modify files directly inside `C:\Program Files\Pure Home\`.

---

## Architecture Overview

```
┌────────────────────────────────────────────────────────────────┐
│  SERVER PC  (one dedicated machine, stays powered on)          │
│                                                                │
│  Tailscale: 100.x.x.x  (stable — never changes)              │
│  PostgreSQL  :5432  ← loopback only                           │
│  WFM Backend :3001  ← accessible via Tailscale network        │
│  Windows Scheduled Task → auto-starts on boot                  │
└────────────────────────────────┬───────────────────────────────┘
                                 │   Tailscale Encrypted Network
            ┌────────────────────┼────────────────────┐
            │                    │                    │
    Employee PC #1       Employee PC #2       Employee PC #3
    Tailscale: 100.x.x.x Tailscale: 100.x.x.x Tailscale: 100.x.x.x
    WFM app installed    WFM app installed    WFM app installed
    C:\Program Files\    C:\Program Files\    C:\Program Files\
    Pure Home\          Pure Home\          Pure Home\
                                 │
                serverUrl = http://100.x.x.x:3001
                (configured once, saved permanently)
```

**Key properties:**
- No public internet exposure — traffic stays within the Tailscale private network
- No Cloudflare tunnels, no temporary URLs, no ngrok
- Server Tailscale IP is stable — does not change after restarts
- All clients use the same URL permanently — no reconfiguration needed
- Works after every PC restart — Tailscale and the backend both auto-start
- Real-time sync via Socket.io — all clients see changes within milliseconds

## Network Requirements

| Requirement | Detail |
|---|---|
| Networking layer | Tailscale (replaces static LAN IP / VPN) |
| Server PC | Tailscale installed, connected, auto-start enabled |
| Employee PCs | Tailscale installed, signed in to same tailnet |
| Port | TCP 3001 inbound on server PC (opened by `configure-firewall.ps1`) |
| Internet | Required only for Tailscale authentication (one-time per device) |
| Bandwidth | Minimal — all traffic is JSON/WebSocket via Tailscale tunnel |

## Part A — Server PC Setup

Run these steps **once**, in order, on the machine designated as the server.

### Step 1 — Install Prerequisites

**PostgreSQL 15 or 16:**
- Download: https://www.postgresql.org/download/windows/
- During setup: set password for `postgres` user, keep port 5432
- Let the installer add PostgreSQL to PATH

**Node.js 20 LTS:**
- Download: https://nodejs.org — choose **"Install for all users"** and **"Add to PATH"**
- Verify after install: open a new PowerShell and run `node --version`

---

### Step 2 — Install and Connect Tailscale (Server PC)

```powershell
# Run as Administrator
.\scripts\install-tailscale-server.ps1
```

This script will:
1. Download and install Tailscale if not already present
2. Open the Tailscale sign-in page
3. Guide you through connecting to the company tailnet
4. Print the server's stable Tailscale IP (e.g. `100.94.5.3`)

**After this step you will have the server's Tailscale IP.**  
Write it down — you will enter this on every employee PC.

**Manual alternative (if the script fails):**
1. Download Tailscale from https://tailscale.com/download/windows
2. Install with default options
3. Click the Tailscale tray icon (system tray, bottom-right) → Sign In
4. Sign in with the company Tailscale account
5. Run `tailscale ip` in PowerShell to find the assigned IP

---

### Step 3 — Install the Backend

```powershell
# 1. Copy project to a permanent server location
robocopy "C:\Users\fahd1\Desktop\WFM-System-Updated" C:\WFM /E /XD node_modules dist out dist-installer backups .git

# 2. Install backend dependencies
cd C:\WFM\packages\backend
npm install

# 3. Generate Prisma client
npm run db:generate

# 4. Create the database
#    Open pgAdmin or run:
$env:PGPASSWORD = "postgres"
& "C:\Program Files\PostgreSQL\16\bin\psql.exe" -U postgres -c "CREATE DATABASE wfm_db;"

# 5. Run migrations
npm run db:migrate

# 6. Seed initial users
npm run db:seed

# 7. Compile backend to production JS
npm run build
```

**Default credentials after seed:**
| Department | Email | Password |
|---|---|---|
| Admin | admin@wfm.local | *(set via seed script)* |
| Scheduling | scheduling@wfm.local | *(set via seed script)* |
| Technician | tech1@wfm.local | *(set via seed script)* |

---

### Step 4 — Configure Firewall

```powershell
# Run as Administrator — opens port 3001 for Tailscale traffic
C:\WFM\scripts\configure-firewall.ps1
```

This creates two Windows Firewall rules:
1. **Tailscale rule**: allows TCP 3001 from `100.64.0.0/10` (the Tailscale CGNAT range)
2. **LAN fallback**: allows TCP 3001 on the Private profile (same-network access)

---

### Step 5 — Configure Auto-Start

```powershell
# Run as Administrator — registers backend as a Windows Scheduled Task
C:\WFM\scripts\install-autostart.ps1
```

The script:
- Builds the backend if `dist/` is missing
- Registers a Scheduled Task named **"WFM Backend"** (runs as SYSTEM at startup)
- Starts the task immediately — no reboot required
- Prints the Tailscale URL to share with employees

**Verify it is running:**
```powershell
Invoke-WebRequest http://127.0.0.1:3001/health | Select-Object -Expand Content
# Expected: {"status":"ok","database":"connected","timestamp":"..."}
```

---

### Step 6 — (Optional) Install Electron App on Server PC

If the admin will also use this machine:
1. Run `Pure Home Setup 1.0.0.exe` as Administrator
2. Open the app → Server Setup → enter `http://127.0.0.1:3001` → Test & Save

## Part B — Employee PC Setup

Repeat for each employee workstation.

### Step 1 — Install Tailscale (Employee PC)

**Option A — Script (run as Administrator):**
```powershell
.\scripts\setup-tailscale-client.ps1
```
The script installs Tailscale, guides sign-in, and tests connectivity to the server.

**Option B — Manual:**
1. Download Tailscale: https://tailscale.com/download/windows
2. Install with default options
3. Click Tailscale tray icon → Sign In → use the **company Tailscale account**
4. Wait for the `100.x.x.x` IP to appear

**All employee PCs must sign in to the same Tailscale account/organisation as the server.**

---

### Step 2 — Install the WFM Application

1. Obtain `Pure Home Setup 1.0.0.exe` from the IT admin or GitHub Release
2. Right-click → **Run as Administrator**
3. Complete the installation wizard
4. The app installs to `C:\Program Files\Pure Home\` (fixed location, not configurable)
5. A desktop shortcut "Pure Home" is created

---

### Step 3 — Configure the Server URL

1. Open **Pure Home** from the desktop shortcut
2. On the department selection screen, click **⚙ Server Setup**
3. Enter the server URL using the Tailscale IP:
   ```
   http://100.x.x.x:3001
   ```
   *(replace `100.x.x.x` with the actual Tailscale IP from Part A Step 2)*
4. Click **Test & Save**
5. Wait for the green "Connected" confirmation
6. Click Back → select your department → enter your access code

**The URL is saved permanently in the app.**  
This only needs to be done once per machine. The Tailscale IP does not change.

---

### Step 4 — Verify Connection

From the employee PC PowerShell:
```powershell
Test-NetConnection -ComputerName 100.x.x.x -Port 3001
# TcpTestSucceeded should be True
```

From the WFM app, a successful login confirms end-to-end connectivity.

## Part C — Backup Procedures

### Automated Daily Backup

Register via PowerShell on the server PC (as Administrator):
```powershell
$action  = New-ScheduledTaskAction -Execute "powershell.exe" `
    -Argument '-ExecutionPolicy Bypass -File "C:\WFM\scripts\backup-database.ps1"'
$trigger = New-ScheduledTaskTrigger -Daily -At "08:00"
Register-ScheduledTask -TaskName "WFM Daily Backup" -Action $action -Trigger $trigger `
    -RunLevel Highest -Force
```

**Backup location:** `C:\WFM\backups\wfm_backup_YYYY-MM-DD_HH-mm-ss.sql`  
**Retention:** Last 14 backups (older files deleted automatically)

### Manual Backup

```powershell
C:\WFM\scripts\backup-database.ps1
```

## Part D — Restore Procedures

```powershell
# 1. Stop the backend
Stop-ScheduledTask -TaskName "WFM Backend"

# 2. Drop and recreate the database
$env:PGPASSWORD = "postgres"
& "C:\Program Files\PostgreSQL\16\bin\psql.exe" -U postgres -c "DROP DATABASE IF EXISTS wfm_db;"
& "C:\Program Files\PostgreSQL\16\bin\psql.exe" -U postgres -c "CREATE DATABASE wfm_db;"

# 3. Restore from backup
& "C:\Program Files\PostgreSQL\16\bin\psql.exe" -U postgres -d wfm_db `
    -f "C:\WFM\backups\wfm_backup_2026-06-07_08-00-00.sql"

# 4. Restart the backend
Start-ScheduledTask -TaskName "WFM Backend"

# 5. Verify
Invoke-WebRequest http://127.0.0.1:3001/health | Select-Object -Expand Content
```

## Part E — Troubleshooting

### Backend not starting

```powershell
# Check task status
Get-ScheduledTask -TaskName "WFM Backend" | Get-ScheduledTaskInfo

# Check if port is in use by another process
netstat -ano | findstr :3001

# Run manually to see error output
cd C:\WFM\packages\backend
node dist/index.js
```

### Employee PC cannot connect to server

```powershell
# Step 1: Verify Tailscale is connected on the employee PC
# (Look for the Tailscale tray icon — it should be green/connected)

# Step 2: Test basic Tailscale reachability
Test-NetConnection -ComputerName 100.x.x.x -Port 3001

# Step 3: Test backend health
Invoke-WebRequest http://100.x.x.x:3001/health | Select-Object -Expand Content

# Step 4: On the server — confirm backend is running
Invoke-WebRequest http://127.0.0.1:3001/health | Select-Object -Expand Content
```

**Common causes:**

| Symptom | Fix |
|---|---|
| "Connection timed out" | Employee PC not connected to Tailscale — click tray icon → Connect |
| "Connection refused" | Backend not running on server — check Scheduled Task, start manually |
| Tailscale icon is grey/disconnected | Sign in to Tailscale: tray icon → Sign In |
| Server IP not known | Run `tailscale ip` on the server PC in PowerShell |
| Wrong server URL in app | Open Pure Home → Server Setup → re-enter the correct 100.x.x.x IP |
| App installed but can't connect | Verify Tailscale sign-in uses the same organisation/account as server |

### Tailscale not connecting after reboot

```powershell
# Check Tailscale service
Get-Service -Name "Tailscale*" | Format-Table Name, Status, StartType

# Start it if stopped
Start-Service -Name (Get-Service -Name "Tailscale*" | Select-Object -First 1).Name
```

### PostgreSQL not starting

```powershell
Get-Service -Name "postgresql*"
Start-Service -Name (Get-Service -Name "postgresql*").Name
```

### Real-time updates not appearing on a client

1. Verify socket connection: look for the connection banner (yellow/red bar at top of app)
2. Reload the application (close and reopen)
3. Verify backend is running: `Test-NetConnection -ComputerName 100.x.x.x -Port 3001`

## Part F — Real-Time Synchronisation Reference

All events propagate via Socket.io within **< 200 ms** on a Tailscale network.

| Action | Event emitted | Who receives it |
|---|---|---|
| Customer created | `customer:created` | All roles |
| Customer updated | `customer:updated` | All roles |
| Customer deleted | `customer:deleted` | All roles |
| Appointment created | `appointment:created` | All roles |
| Appointment status change | `appointment:status` | All roles |
| Task approved / assigned | `task:approved` | Technician + Scheduling |
| Task started | `task:approved` | All roles |
| Task completed | `task:completed` | All roles |
| Task postponed | `task:postponed` | All roles |
| New notification | `notification:new` | All roles |
| New direct message | `dm:new` | Recipient role only |
| Message deleted | `dm:deleted` | Relevant roles |

**Polling fallback:** React Query refetches all data every 30 seconds as a safety net for any missed socket events.

## Part G — Updates

### Electron App Update (Employee PCs)

The app checks GitHub Releases automatically on startup (via `electron-updater`).

**When an update is available:**
1. A notification banner appears in the app
2. The user clicks "Download Update"
3. After download, they click "Install & Restart"
4. Windows UAC prompts for Administrator credentials (required for `C:\Program Files\`)
5. The installer runs elevated, updates the app, and relaunches

**Manual update (if auto-update is unavailable):**
1. Download the new `Pure Home Setup x.x.x.exe` from the GitHub Release
2. Right-click → **Run as Administrator**
3. The installer updates in-place; server URL is preserved

### Backend Update (Server PC)

```powershell
# 1. Stop the running task
Stop-ScheduledTask -TaskName "WFM Backend"

# 2. Copy updated backend files from the source
robocopy "C:\Users\fahd1\Desktop\WFM-System-Updated\packages\backend" `
    "C:\WFM\packages\backend" /E /XD node_modules dist

# 3. Install new dependencies
cd C:\WFM\packages\backend
npm install

# 4. Run new migrations (if any)
npm run db:migrate

# 5. Rebuild
npm run build

# 6. Restart
Start-ScheduledTask -TaskName "WFM Backend"

# 7. Verify
Invoke-WebRequest http://127.0.0.1:3001/health | Select-Object -Expand Content
```

### Building a New Release

```batch
REM From the source directory on the developer PC:
cd C:\Users\fahd1\Desktop\WFM-System-Updated
scripts\build-release.bat
```

Then upload these files to a new GitHub Release (tag `v1.x.x`):
- `packages\unified-app\dist-installer\Pure Home Setup 1.x.x.exe`
- `packages\unified-app\dist-installer\latest.yml`

Stop-ScheduledTask -TaskName "WFM Backend"
